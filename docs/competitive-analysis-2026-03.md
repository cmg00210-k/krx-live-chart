## A. 로딩 성능

### A-1. First Contentful Paint (FCP)

| 플랫폼 | 추정 FCP | 근거 |
|--------|---------|------|
| **CheeseStock** | ~1.2-1.8s | 정적 HTML 즉시 렌더링, `app-loading-overlay` 스피너 0ms 표시. CDN preconnect 4개 (`jsdelivr`, `fonts.googleapis`, `fonts.gstatic`, `unpkg`). CSS 1개 파일(~44KB). Cloudflare Pages 엣지 서빙. |
| **Investing.com** | ~1.5-2.5s | 무거운 광고 스크립트, 다수 third-party 트래커. 하지만 글로벌 CDN(Cloudflare Enterprise) 활용. |
| **키움 영웅문 웹** | ~3-5s | ActiveX/OCX 의존 레거시. 웹 버전은 실질적으로 제한적이며 HTS 설치 유도. |
| **삼성 POP 웹** | ~3-5s | 웹트레이딩은 있으나 HTS 다운로드 중심 아키텍처. Java/ActiveX 잔재. |
| **NH 나무** | ~2-3s | MTS 중심, 웹 버전은 제한적. 나무X(2026 하반기) 출시 시 개선 예상. |
| **한투 뱅키스** | ~3-4s | 웹 트레이딩 있으나 HTS 의존도 높음. |
| **TradingView** | ~0.8-1.2s | 최고 수준 최적화. 자체 차트 엔진, 코드 스플리팅, 점진적 로딩. |

**CheeseStock 평가: ★★☆ (Comparable)**

**강점**:
- `defer` 속성으로 16개 JS 파일 병렬 다운로드 + 순서 실행 보장
- `<link rel="prefetch" href="data/index.json">`으로 종목 데이터 선행 로드
- Service Worker(Cache-First 정적 자산)로 재방문 시 거의 즉시 렌더링
- 빌드 시스템 없음 = 번들링 오버헤드 없음 (역설적 장점)
- 로딩 오버레이가 JS 실행 전 HTML에 직접 포함 → 체감 FCP 0ms

**약점**:
- 16개 개별 JS 파일 = 16번 HTTP 요청 (HTTP/2 멀티플렉싱으로 완화되지만, 번들링 대비 비효율)
- Lightweight Charts CDN (`unpkg.com`)이 단일 장애점 (SW 캐시로 2차 방문 이후 해소)
- Pretendard + JetBrains Mono 웹폰트 2종 = FOIT/FOUT 가능성

### A-2. Time to Interactive (TTI)

| 플랫폼 | 추정 TTI | 근거 |
|--------|---------|------|
| **CheeseStock** | ~2.5-4s | `init()` → `initFromIndex()` → 첫 종목 `getCandles()` → `chartManager.create()` 완료까지. WS 프로브 비동기화로 3초 블로킹 제거됨. |
| **TradingView** | ~1.5-2s | 점진적 로딩, 차트 우선 렌더 후 지표/툴 지연 로드 |
| **Investing.com** | ~3-5s | 광고/트래커가 메인 스레드 점유 |
| **국내 증권사 웹** | ~5-10s+ | 인증 과정 + 보안 모듈 로드 + OCX 초기화 |

**CheeseStock 평가: ★★☆ (Comparable to global, superior to Korean brokerages)**

**핵심 최적화 구조 분석**:
```
HTML 파싱 ──→ 로딩 오버레이 즉시 표시 (체감 FCP ~0ms)
    │
    ├─ defer JS 16개 병렬 다운로드 시작
    ├─ prefetch data/index.json 선행 로드
    ├─ preconnect CDN 4개 DNS/TLS 선행
    │
    ▼
init() 시작
    │
    ├─ WS 프로브: 비동기 백그라운드 (블로킹 없음!)
    │   └─ 성공 시 ws 모드 복원, 실패 시 file 모드 유지
    │
    ├─ initFromIndex(): index.json 로드 → ALL_STOCKS 2733종목
    │
    ├─ getCandles(): L1(메모리) → L2(IndexedDB) → L3(네트워크)
    │   └─ IDB 캐시 히트 시 네트워크 요청 없이 즉시 반환
    │
    ├─ chartManager.create() → 차트 렌더링
    │
    └─ 로딩 오버레이 fade-out (400ms 트랜지션)
```

**개선 기회**:
- index.json `prefetch` → `preload`로 변경 시 더 빠른 로드 가능 (우선순위 상승)
- 핵심 JS(colors.js + api.js + chart.js)를 inline 또는 `<link rel="preload" as="script">`
- Lightweight Charts를 self-host하면 unpkg.com 의존성 제거 가능

| 플랫폼 | 추정 시간 | 비고 |
|--------|----------|------|
| **CheeseStock** | ~200-500ms (캐시) / ~1-2s (네트워크) | `_selectVersion`으로 stale fetch 무시. Web Worker 분석은 별도 스레드. |

4. **file 모드 신선도**: T+1 (어제 데이터). `daily_update.bat` 매일 16:00 실행 필요.
5. **demo 모드**: 시뮬레이션 데이터 — 투자 판단에 사용 불가

**경쟁사 대비**:

**CheeseStock 강점**: 3단계 graceful degradation은 경쟁 대비 매우 우수한 설계. 데이터 출처 추적(`_dataSource` 태그) + 워터마크("데모 — 실제 데이터 아님") + 시드 데이터 경고 배너는 사용자 신뢰 측면에서 모범적.

## 종합 SWOT 분석

### Strengths (강점)
1. **무료 + 비로그인**: 계좌 개설 없이 접근 가능한 유일한 한국 주식 기술적 분석 플랫폼
2. **패턴 엔진 독보적**: 26패턴 자동 감지 + 품질 점수 + ATR 정규화 + Forecast Zone = TradingView Premium($678/yr)급 기능을 무료 제공
3. **WLS 회귀 백테스트**: 패턴 기반 수익률 예측 + 95% CI + 승률/기대수익 — 타 플랫폼에 없는 고유 기능
4. **복합 시그널**: 16개 개별 + 6개 복합 시그널 자동 매칭 + 다이버전스 감지
5. **한국식 색상 완벽**: `KRX_COLORS` 중앙 관리 시스템
6. **3-tier 캐싱**: 메모리 → IndexedDB → 네트워크 + Service Worker 오프라인
7. **Web Worker**: 분석 엔진 별도 스레드 → 메인 스레드 차트 상호작용 비차단
8. **학술 기반**: 16개 core_data 학술 문서 + 5개 구현 문서로 뒷받침된 수학적 엄밀성

### Weaknesses (약점)
1. **데이터 인프라**: Kiwoom OCX 1대 PC 의존 = 단일 장애점, 확장 불가
2. **지표 부족**: 13종 vs TradingView 100+ vs 키움 수백 종
3. **드로잉 도구 부족**: 6종 vs TradingView 110+
4. **차트 타입 부족**: 4종 vs TradingView 20+
5. **재무 데이터 깊이**: DART 미연동 시 추정치. 컨센서스/배당/현금흐름 없음.
6. **모바일 한계**: 전용 앱 없음. 드로잉 도구 모바일 숨김. PWA manifest 미구현.
7. **사용자 계정 없음**: 설정/드로잉 동기화 불가 (localStorage 한정)
8. **뉴스/커뮤니티 없음**: 종목 관련 뉴스, 애널리스트 리포트, 종목토론 미제공
9. **분봉 데이터 품질**: file 모드 분봉은 Brownian bridge 보간 = 실제 데이터 아님
10. **라이트 테마 없음**: 다크 온리

### Opportunities (기회)
1. **Koscom API 전환**: 안정적 데이터 인프라 확보 → 공개 서비스 가능
2. **PWA 전환**: manifest.json + 앱 설치 유도 → 네이티브 앱 수준 UX
3. **사용자 계정 + 클라우드 동기화**: Firebase/Supabase로 설정/드로잉 동기화
4. **AI 분석 통합**: GPT 기반 차트 해석 (Investing.com, NH 나무 이미 도입 중)
5. **커뮤니티 기능**: 종목 토론, 공유 차트, 패턴 알림
6. **ETF/해외 주식 확장**: 현재 KRX 전용 → 미국 주식 확장 시 TAM 확대

### Threats (위협)
1. **TradingView 한국어 강화**: 이미 kr.tradingview.com 운영. 한국 사용자 증가 추세.
2. **국내 증권사 웹 고도화**: NH 나무X(2026 하반기) 크로스플랫폼 + AI 차트분석
3. **Investing.com AI**: Vision AI 기반 자동 차트 분석 — 기존 패턴 엔진 대비 유연성 높음
4. **Kiwoom OCX 정책 변경**: OCX 접속 제한 강화 시 데이터 파이프라인 단절 위험
5. **무료 모델 지속성**: 서버 운영비(Cloudflare Tunnel, PC 상시 가동) 수익 없이 부담
