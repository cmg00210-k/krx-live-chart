Part 2. 미시적 측면 — 패턴 인식과 시각화 시스템

Chapter 1. 캔들스틱 패턴 (21종)

CheeseStock의 js/patterns.js는 1,843줄 규모의 PatternEngine 클래스로 21종의 캔들스틱 패턴을 감지한다. 모든 패턴은 ATR(Average True Range) 정규화를 기반으로 변동성에 무관한 일관된 감지 성능을 보장하며, 5요인 품질 점수(body 25%, volume 25%, trend 20%, shadow 15%, extra 15%)를 산출한다 [Nison, 1991], [Morris, 2006].

감지 조건 (patterns.js:detectHammer(), line 310):

  1. body > range * MIN_BODY_RANGE (0.10) — 실체 존재 확인

매수 중간 Bulkowski 승률: 60% [Bulkowski, 2005]

        ╹

감지 조건 (patterns.js:detectDoji(), line 486):

중립 약함 Bulkowski 승률: 51% [Bulkowski, 2005]

차트 패턴은 스윙 포인트(swing high/low) 기반으로 감지된다. PatternEngine의 _findSwingHighs(candles, lookback=3)와 _findSwingLows(candles, lookback=3) 메서드가 좌우 3봉 비교로 극값을 찾고, 이를 기반으로 9종의 차트 패턴을 인식한다. 모든 차트 패턴은 Bulkowski의 "measured move" 공식으로 목표가를 계산한다 [Bulkowski, 2005].

8.3 주요 방어 패턴

8.3.1 Null/Undefined 가드

모든 패턴 감지 함수는 입력 유효성을 선검증한다:

  • analyze(): if (!candles || candles.length < 10) return [];
  • signalEngine.analyze(): if (!candles || candles.length < 30) return { signals: [], ... };
  • 모든 지표 함수: 빈 배열/짧은 입력 시 null 배열 반환

8.3.2 ATR Fallback

ATR_FALLBACK_PCT = 0.02: ATR이 null인 경우 가격의 2%를 대체값으로 사용. 모든 ATR 참조에서 일관되게 적용.

8.3.3 Look-Ahead Bias 방지

교수형, 잉태형에서 candles[i+1] 미래 참조를 삭제하고, 확인 캔들 없는 상태를 기본값으로 보수적 평가 (교수형: extra=0.15, 잉태형: quality*0.8).

8.3.4 캐싱 전략 (3-Layer)

  1. IndicatorCache (indicators.js): Lazy evaluation. 각 지표 최초 접근 시 계산, setCandles() 또는 invalidate()로 무효화.
  2. Chart Indicator Cache (chart.js): candles.length + lastTime + lastClose 키로 O(n) 재계산 방지.
  3. Worker Analysis Cache (analysisWorker.js): 캔들 지문으로 pattern/signal 결과 캐싱. 드래그 반복/종목 재선택 시 재분석 스킵.

8.3.5 중복 제거

PatternEngine._dedup(): `${type}-${endIndex}` 키로 Set 기반 중복 필터링. 복합 시그널: compositeId + ±window 범위 내 중복 방지.

8.4 종합 품질 등급: A

87개 try/catch, 1,005+건 가드 절, look-ahead bias 방지, 3-layer 캐시, ATR fallback, 중복 제거를 갖춘 방어적 코드베이스. 모든 패턴이 학술 문헌에 기반한 상수를 사용하며, 매직 넘버가 완전히 제거되었다. 프론트엔드 전용 아키텍처에서 이 수준의 코드 품질은 상용 수준으로 평가된다.

C.1 지지/저항 수준 탐지

  1. 모든 스윙 고점(resistance)과 저점(support)을 수집
  2. 가격 허용오차 = lastATR * 0.5 내에서 클러스터링
  3. 2회 이상 터치된 클러스터만 유효

부록 E. KRX 시장 특수성 대응

E.1 가격제한폭 (30%) 대응

KRX 시장은 일일 가격제한폭이 ±30%이므로, 국제 시장 대비 갭(gap)이 제한적이다. 이에 따른 조정 사항:

  • ENGULF_BODY_MULT: 국제 1.3x → KRX 1.2x (장악 배율 하향)
  • 관통형/먹구름형: 갭 조건을 "이하/이상"으로 완화 (strict gap 대신 동일 수준 허용)
  • 샛별/석별: 갭 조건에서 별(2봉) 위치를 전봉 종가 대비 확인 (hard gap 미요구)

E.2 한국식 색상 체계

한국 증시는 상승=빨강, 하락=파랑을 사용한다 (미국/유럽은 상승=초록, 하락=빨강). 이는 KRX_COLORS.UP = '#E05050', KRX_COLORS.DOWN = '#5086DC'으로 구현되어 있으며, 차트의 모든 방향 표시(캔들, 수익률, 변동폭)에 일관 적용된다.

E.3 KST 타임존 처리

일봉: "YYYY-MM-DD" 문자열 (timezone 이슈 없음). 분봉: Unix timestamp where 09:00 KST = 00:00 UTC. chart.js의 tickMarkFormatter와 timeFormatter가 UTC → KST 변환을 수행하며, string/businessDay/timestamp 3가지 형식을 모두 처리한다.

E.4 시장 시간

정규장: 09:00~15:30 KST. 프리마켓: 08:00. 애프터마켓: 15:30~16:00. 분봉 데이터 생성(generate_intraday.py)에서 이 시간대를 준수하며, calendar.timegm()으로 UTC 타임스탬프를 생성하여 KST 기반 datetime.timestamp()의 이중 감산 문제를 방지한다.
