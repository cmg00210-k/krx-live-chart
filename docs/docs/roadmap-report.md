# Part 1: 기술적 패턴 구현 과정

## "주식 병원"을 짓다

CheeseStock이 만든 것을 이해하려면, **병원**을 짓는 과정으로 비유하면 됩니다.

주식 시장에 참여하는 사람들은 매일 수많은 질문을 합니다:
- "이 주식, 지금 사도 되나?"
- "이 차트 모양이 뭘 의미하지?"
- "과거에 이런 모양이 나왔을 때 결과가 어땠나?"

CheeseStock은 이 질문들에 **데이터와 학술 근거로** 답하는 시스템입니다.

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│           CheeseStock = "주식 기술적 분석 병원"              │
│                                                             │
│   환자(주식)가 들어오면 → 검사하고 → 진단하고 → 결과를 보여준다  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1층: 진단 장비실 -- patterns.js (1,488줄)

**비유**: MRI, CT, X-ray 같은 의료 장비

병원에 가면 의사가 눈으로만 보지 않습니다. MRI를 찍고, 혈액검사를 하고, 여러 장비로 정밀 검사를 합니다.

`patterns.js`는 주식 차트에서 **26종의 패턴**을 자동으로 찾아내는 "진단 장비"입니다.

```
    ┌─────────────── 26종 패턴 감지 엔진 ───────────────┐
    │                                                    │
    │  [캔들스틱 패턴 17종]          [차트 패턴 8종]       │
    │                                                    │
    │   해머          도지            상승삼각형           │
    │   역해머        잠자리도지       하락삼각형           │
    │   교수형        비석도지        상승쐐기             │
    │   유성형        적삼병          하락쐐기             │
    │   상승장악      흑삼병          이중바닥             │
    │   하락장악      관통형          이중천장             │
    │   상승잉태      먹구름          H&S (머리어깨)       │
    │   하락잉태      족집게바닥       역H&S              │
    │   샛별형        족집게천장                          │
    │   석별형                                           │
    │                                                    │
    │   + 지지/저항선 자동 감지 (1종)                      │
    │                                                    │
    └────────────────────────────────────────────────────┘
```

**무엇이 특별한가?**

일반적인 차트 앱은 "봉이 이렇게 생겼으면 해머"라고 단순하게 판단합니다.
CheeseStock은 다릅니다:

| 검사 항목 | 설명 | 비유 |
|-----------|------|------|
| ATR 정규화 | 변동성 대비 상대적 크기 측정 | 키 180cm인 사람과 160cm인 사람의 "큰 손"은 다름 |
| 거래량 확인 | 패턴 발생 시 거래량이 뒷받침되는지 | 기침만 하면 감기, 피까지 나오면 다른 병 |
| 추세 맥락 | 하락 추세 끝 해머 vs 상승 중 해머 구분 | 같은 증상도 환자 이력에 따라 진단이 다름 |
| 품질 점수 | body(25%) + volume(25%) + trend(20%) + shadow(15%) + extra(15%) | 5가지 항목 종합 건강 점수 |

각 패턴에는 Steve Nison(1991), Thomas Bulkowski(2005), Gregory Morris(2006) 등
금융 학술 문헌에서 검증된 임계값을 사용합니다.

**예시 -- 해머(Hammer) 패턴 감지 과정**:

```
    가격
     │
     │   ┌──┐
     │   │  │  ← 작은 몸통 (body/range < 일정 비율)
     │   └──┘
     │     │
     │     │   ← 긴 아래꼬리 (body의 2배 이상)
     │     │
     │     │
     ─────┴───── 시간

    검증 단계:
    [1] 아래꼬리가 몸통의 2배 이상인가?           --> SHADOW_BODY_MIN = 2.0
    [2] 윗꼬리가 거의 없는가?                    --> COUNTER_SHADOW_MAX = 0.15
    [3] 직전 추세가 하락이었는가?                 --> _detectTrend() < -0.3
    [4] ATR 대비 유의미한 크기인가?               --> range > MIN_RANGE_ATR * ATR
    [5] 거래량이 평균 이상인가?                   --> volume > MA(20)
    [6] 품질 점수 = (body 0.25 + vol 0.25 + trend 0.20 + shadow 0.15 + extra 0.15)
    [7] 목표가 = 현재가 + ATR * 1.5 (Bulkowski measured move)
    [8] 손절가 = 저가 - ATR * 2.0
```

### 2층: 의사의 판단 시스템 -- signalEngine.js (1,129줄)

**비유**: 검사 결과를 종합해서 "수술할지, 약 먹을지" 결정하는 전문의

진단 장비(patterns.js)가 "해머 패턴 발견!"이라고 알려줘도,
그것만으로 "사세요"라고 말하면 무책임합니다.

의사는 여러 검사 결과를 **종합적으로** 판단합니다:
- MRI에서 이상 소견 + 혈액검사 정상 → "관찰"
- MRI에서 이상 소견 + 혈액검사 이상 + 초음파 이상 → "치료 필요"

signalEngine.js는 이와 동일하게 **16개 개별 시그널**을 5개 카테고리로 분류하고,
여러 시그널이 동시에 발생할 때 **6개 복합 시그널**로 묶어 판단합니다.

```
    ┌─────────────── 시그널 엔진 구조 ───────────────────┐
    │                                                     │
    │  [5개 카테고리 x 16개 개별 시그널]                    │
    │                                                     │
    │  1. 이동평균 크로스     골든크로스 / 데드크로스        │
    │  2. RSI 과매수/과매도   RSI 진입 / RSI 탈출          │
    │  3. MACD 시그널        골든/데드 크로스, 제로 크로스  │
    │  4. 볼린저 밴드         밴드 터치 / 이탈             │
    │  5. 거래량 이상         거래량 급증 / 투매 감지       │
    │                                                     │
    │           ┌─── 복합 시그널 조합 ───┐                │
    │           │                        │                │
    │  Tier 1:  │ 해머 + RSI과매도탈출   │  → 강력매수    │
    │  (강력)   │ + 거래량급증           │    신뢰도 82%  │
    │           │                        │                │
    │  Tier 2:  │ 골든크로스 + RSI       │  → 매수       │
    │  (중간)   │                        │    신뢰도 72%  │
    │           │                        │                │
    │  Tier 3:  │ BB하단 + 거래량        │  → 관심 매수   │
    │  (약한)   │                        │    신뢰도 62%  │
    │           └────────────────────────┘                │
    │                                                     │
    │  + 다이버전스 감지 (가격 vs RSI/MACD 괴리)           │
    │  + 시장 심리 종합 점수 (sentiment scoring)            │
    │                                                     │
    └─────────────────────────────────────────────────────┘
```

**핵심 개념 -- "컨플루언스(Confluence)"**:

혼자 "해머 패턴!"이라고 외치면 믿음직하지 않습니다.
하지만 동시에 RSI가 과매도에서 탈출하고, 거래량이 터지고, MACD가 골든크로스를 보이면?
이것이 **컨플루언스** -- 여러 독립적 근거가 한 방향을 가리키는 것입니다.

이것은 마치 **3명의 전문의가 동시에 같은 진단을 내리는 것**과 같습니다.

### 3층: 치료 성공률 통계실 -- backtester.js (517줄)

**비유**: "이 수술의 성공률은 82%입니다" -- 과거 기록 기반 통계

의사가 "해머 패턴이니 매수하세요"라고 했을 때, 환자는 묻습니다:
**"이 치료, 성공률이 어떻게 되나요?"**

backtester.js는 과거 데이터에서 같은 패턴이 나왔을 때
**실제로 며칠 후 얼마나 올랐는지/내렸는지**를 통계로 보여줍니다.

```
    ┌─────────────── 백테스팅 엔진 ──────────────────────┐
    │                                                     │
    │  입력: 과거 주가 데이터 (OHLCV 캔들)                 │
    │                                                     │
    │  처리:                                              │
    │  [1] 전체 기간에서 패턴 발생 지점 스캔               │
    │  [2] 각 발생 후 1일/3일/5일/10일/20일 수익률 계산    │
    │  [3] 승률, 평균수익, 최대손실, 리스크리워드 산출      │
    │  [4] WLS 회귀로 "이번 패턴의 품질"로 수익 예측       │
    │                                                     │
    │  출력 예시:                                         │
    │  ┌──────────────────────────────────────────┐       │
    │  │ 해머 패턴 (최근 1년)                       │       │
    │  │                                          │       │
    │  │  발생 횟수:  12회                         │       │
    │  │  5일 승률:   75.0% (9승 3패)              │       │
    │  │  평균 수익:  +2.3%                        │       │
    │  │  평균 손실:  -1.1%                        │       │
    │  │  R:R 비율:  2.09 : 1                     │       │
    │  │  최대 수익:  +5.8%                        │       │
    │  │  최대 손실:  -2.4%                        │       │
    │  │                                          │       │
    │  │  WLS 예측: +1.8% (95% CI: -0.5% ~ 4.1%) │       │
    │  └──────────────────────────────────────────┘       │
    │                                                     │
    │  * KRX 거래비용 0.36% (수수료+세금) 반영             │
    │  * Look-ahead bias 제거 (다음봉 시가 기준 진입)      │
    │                                                     │
    └─────────────────────────────────────────────────────┘
```

**WLS 회귀 예측이란?**

보통 백테스트는 "평균 수익률"만 알려줍니다.
CheeseStock은 한 걸음 더 나갑니다.

WLS(Weighted Least Squares, 가중최소제곱)는 **"이번에 발생한 이 해머의 품질이
어느 정도이고, 현재 추세와 거래량이 어떤지를 고려했을 때,
이번 해머는 특히 +2.3% 정도 오를 것 같다"** 라고 예측하는 것입니다.

이것은 마치 의사가 "이 수술의 평균 성공률은 80%인데,
**당신은 나이가 젊고 체력이 좋으니 90% 정도**입니다"라고
개인 맞춤형 예측을 해주는 것과 같습니다.

```
    WLS 회귀 모델:

    예상 수익률 = b0 + b1*(패턴 품질) + b2*(추세 강도) + b3*(거래량 비율) + b4*(변동성)

    가중치 w_i = 1 / sigma_i^2   (시간이 오래된 관측일수록 가중치 낮춤)

    95% 예측 구간 = 예측값 +- t(0.025, n-p) * sqrt(MSE * (1 + x' * (X'WX)^-1 * x))
```

95% 신뢰구간(CI)까지 제공하므로, "최악의 경우 -0.5%, 최선의 경우 +4.1%"처럼
불확실성의 범위를 보여줍니다. Bloomberg 터미널에도 없는 기능입니다.

### 4층: 환자에게 보여주는 검사 결과 화면 -- patternRenderer.js (1,400+줄)

**비유**: 환자가 이해하기 쉬운 검사 결과지 (컬러 이미지, 수치, 정상/비정상 표시)

의사가 아무리 정확하게 진단해도, 환자에게 **영어로 된 의학 논문**을 보여주면 의미가 없습니다.
검사 결과를 알기 쉬운 화면으로 보여줘야 합니다.

patternRenderer.js는 증권사 HTS(Home Trading System) 수준의 **전문 시각화**를 제공합니다.

```
    ┌────────────────── 9층 렌더링 구조 ──────────────────┐
    │                                                      │
    │  Canvas2D ISeriesPrimitive 기반 직접 그리기           │
    │                                                      │
    │  Layer 1: 글로우 효과      (패턴 영역 은은한 빛)      │
    │  Layer 2: 브래킷           (패턴 범위 대괄호)         │
    │  Layer 3: 추세 영역        (삼각형/쐐기 채우기)       │
    │  Layer 4: 폴리라인         (추세선, 넥라인)           │
    │  Layer 5: 수평선           (지지/저항, 목표/손절)      │
    │  Layer 6: 연결선           (패턴 포인트 간 연결)       │
    │  Layer 7: 라벨             (패턴명 pill badge)       │
    │  Layer 8: Forecast Zone    (목표/손절 예측 영역)      │
    │  Layer 9: 연장선           (오프스크린 구조선 확장)    │
    │                                                      │
    │  색상 체계:                                          │
    │   매수 = 민트/시안 (PTN_BUY)                         │
    │   매도 = 라벤더/퍼플 (PTN_SELL)                      │
    │   구조선 = 금색 (PTN_STRUCT)                         │
    │   손절 = PTN_STOP, 목표 = PTN_TARGET                 │
    │                                                      │
    │  R:R 비율 바 (Autochartist/TrendSpider 벤치마크):     │
    │  ┌───────────────────────────────────────────┐       │
    │  │ ████████████  R:R 2.1:1  ○○○○○○          │       │
    │  │ (녹색=목표)              (적색=손절)       │       │
    │  └───────────────────────────────────────────┘       │
    │                                                      │
    │  Forecast Zone (목표가 영역):                         │
    │  ┌───────────────────────────────────────────┐       │
    │  │  ░░░░░░░░░░░ +3.2% 목표 ░░░░░░░░░░░      │       │
    │  │  ─────────── 현재가 ──────────────        │       │
    │  │  ╲╲╲╲╲╲╲╲╲╲ -1.5% 손절 ╲╲╲╲╲╲╲╲╲╲      │       │
    │  └───────────────────────────────────────────┘       │
    │  (목표: 부드러운 그라데이션 / 손절: 사선 줄무늬)       │
    │                                                      │
    │  3-tier 가시성 필터링:                                │
    │   캔들 패턴 → 화면 내 봉만 표시                       │
    │   차트 패턴 → 오프스크린도 구조선 연장                 │
    │   지지/저항 → 항상 표시                               │
    │                                                      │
    │  라벨 충돌 방지 + 최대 5개 패턴 (신뢰도순 정렬)       │
    │                                                      │
    └──────────────────────────────────────────────────────┘
```

### 응급실: 실시간 모니터링 -- realtimeProvider.js + WebSocket 서버

**비유**: 중환자실(ICU)의 심전도 모니터 -- 실시간으로 환자 상태를 추적

일반 병원은 진찰 시간에만 환자를 봅니다.
응급실은 **24시간 실시간**으로 심전도, 혈압, 산소포화도를 모니터링합니다.

CheeseStock의 실시간 시스템도 마찬가지입니다:

```
    ┌──── Kiwoom OCX 서버 ──── (seth1 PC) ──────────────┐
    │                                                     │
    │  Python 3.9 (32-bit) + PyQt5                       │
    │  ┌─────────────────────────────────┐               │
    │  │  Kiwoom OpenAPI+ OCX (QAxWidget)│               │
    │  │  ┌───────────────────────┐      │               │
    │  │  │ OnReceiveRealData     │──┐   │               │
```

### 약국: 3계층 캐싱 시스템 -- api.js (420줄)

**비유**: 약을 매번 제약회사에서 가져오면 느리니까,
약국(L2) → 병원 창고(L1) → 환자 손(메모리)에 미리 보관해두는 것

```
    ┌────────── 3-Tier 캐싱 아키텍처 ──────────────┐
    │                                               │
    │  L1. 메모리 캐시 (JavaScript Map)             │
    │      → 즉시 접근 (0ms)                        │
    │      → 페이지 새로고침 시 소멸                 │
    │                                               │
    │  L2. IndexedDB 캐시 (브라우저 영구 저장)       │
    │      → 빠른 접근 (~5ms)                       │
    │      → 페이지 새로고침 후에도 유지             │
    │      → 종목코드-타임프레임별 저장              │
    │                                               │
    │  L3. 네트워크 요청                             │
    │      → WebSocket: 실시간 (100~500ms)          │
    │      → File: JSON fetch (~200ms)              │
    │      → Demo: 즉시 생성 (0ms)                  │
    │                                               │
    │  조회 순서: L1 → L2 → L3 (Cache-Aside 패턴)  │
    │  L3에서 받으면 L1 + L2에 동시 저장             │
    │                                               │
    └───────────────────────────────────────────────┘
```

### 기술 지표 계산 엔진 -- indicators.js (888줄)

| 지표 | 무엇을 측정하나 | 비유 |
|------|-----------------|------|
| MA (이동평균) | 최근 N일 평균 가격 | 체온 (평균 기준) |
| EMA (지수이동평균) | 최근 가격에 더 가중치 | 최근 체온 변화 추이 |
| 볼린저 밴드 | 가격 변동 범위 | 정상 혈압 범위 |
| RSI | 상승/하락 힘의 비율 | 맥박 -- 너무 빠르거나 느린지 |
| MACD | 추세 방향/강도 | 건강 추세 (점점 좋아지나/나빠지나) |
| ATR | 일일 변동 폭 | 환자의 활동량 |
| 일목균형표 | 5개 선으로 추세/지지/저항 종합 | 종합 건강검진표 |
| 칼만 필터 | 노이즈 제거된 가격 추정 | 오차를 줄인 정밀 체온 |
| 허스트 지수 | 추세 지속 가능성 | 회복세가 계속될지 예측 |

**Koscom 계약이 왜 중요한가?**

주식 데이터는 "공짜"가 아닙니다. 한국거래소(KRX)가 생산하는 데이터는
Koscom(코스콤)이 독점 유통합니다.

```
    KRX (한국거래소)
         │
         │  데이터 생산
         ▼
    Koscom (코스콤) ──── 유일한 공식 유통사
         │
         ├── 증권사 (키움, 삼성, 미래에셋 등)
         ├── 포털 (네이버, 다음)
         ├── 핀테크 (증권플러스, 토스증권)
         └── CheeseStock (우리가 여기에 계약해야 함)

    현재 상태:
    키움증권 OCX → 개인용 API → 재배포 법적 회색지대

    목표 상태:
    Koscom API → 정보분배계약 체결 → 합법적 재배포
```

## 마무리: 이 프로젝트의 의미

CheeseStock은 **2명의 개발자가 0원의 초기 비용으로** 만든 프로젝트입니다.

코드 11,000줄. 16개 JavaScript 모듈. 26종 패턴 감지. WLS 회귀 예측.
HTS급 Canvas 시각화. 실시간 WebSocket. 학술 논문 수준 메타데이터.

TradingView는 400명이 10년 걸려 만들었습니다.
증권플러스는 50명이 5년 걸려 만들었습니다.

우리는 2명이 시작합니다. 모든 것을 만들 수는 없지만,
**"KRX 기술적 패턴 분석"이라는 좁은 영역에서 최고**가 될 수 있습니다.

```
    ┌───────────────────────────────────────────────┐
    │                                               │
    │  "넓고 얕게"가 아니라 "좁고 깊게"              │
    │                                               │
    │  TradingView = 세계 모든 시장의 모든 것        │
    │  CheeseStock = 한국 주식의 패턴 분석 하나      │
    │                                               │
    │  좁은 영역의 1등이 넓은 영역의 꼴등보다 낫다    │
    │                                               │
    └───────────────────────────────────────────────┘
```

가장 중요한 다음 한 걸음은 **Phase 1의 패턴 정확도 추적 시스템**입니다.
"우리가 감지한 패턴이 실제로 맞았는가?"에 대한 데이터가 쌓이는 순간,
CheeseStock은 "재미있는 사이드 프로젝트"에서 "신뢰할 수 있는 분석 도구"로 전환됩니다.
