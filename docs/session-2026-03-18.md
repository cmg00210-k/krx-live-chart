## 오늘 한 작업 (순서대로)

### 1. HTTPS 설정 완료
- Lightsail 방화벽에 HTTPS(443) 포트 추가 (IPv4 + IPv6)
- certbot으로 Let's Encrypt 인증서 발급
  ```bash
  sudo certbot --nginx -d cheesestock.co.kr
  ```
- www 인증서는 DNS 레코드 없어서 실패 → apex 도메인만 발급

### 2. GitHub origin/main 머지 + 공개 서버 버그 수정
- **버그**: 공개 서버에서 WS 연결 가이드 팝업이 떠서 화면이 막히는 문제
- **원인**: `app.js`에서 WS 실패 시 `_showConnectionGuide()` 무조건 호출
- **수정**: `_isLocalDev` 체크 추가 → localhost에서만 팝업, 공개 서버는 자동 file 모드

```javascript
// js/app.js (477~481줄)
var _isLocalDev = (window.location.hostname === 'localhost' ||
                   window.location.hostname === '127.0.0.1' ||
                   window.location.protocol === 'file:');
if (_originalMode === 'ws' && KRX_API_CONFIG.mode !== 'ws' && _isLocalDev) {
    _showConnectionGuide(function() { _continueInit(); });
    return;
}
```

### 3. 데이터 없음 문제 해결
- **버그**: 종목 클릭 시 "데이터 없음" 빈 차트
- **원인**: `data/` 폴더가 `.gitignore`에 있어서 서버에 JSON 파일 없음
- **수정 1**: `api.js` — file 모드에서 JSON 없으면 demo 폴백 추가
- **수정 2**: `.gitignore` 수정 — `data/kospi/`, `data/kosdaq/` git 포함 허용
- **수정 3**: KOSPI 35개 + KOSDAQ 20개 실제 데이터 git에 커밋

### 4. www 도메인 오류 수정
- **버그**: `index.html`, `sitemap.xml`, `robots.txt`가 `www.cheesestock.co.kr`을 가리킴
- **원인**: 팀원이 www 기준으로 작성 (www DNS 레코드 없음, 인증서도 없음)
- **수정**: 3개 파일 모두 `cheesestock.co.kr`으로 변경

### 5. 문서 현행화
- `docs/deployment.md`: Vercel → AWS Lightsail 내용으로 전면 재작성
- `docs/developer-setup.md`: 온보딩 가이드 최신 상태 반영

## 오늘 발생한 충돌/오류 기록

| 오류 | 원인 | 해결 |
|------|------|------|
| ERR_CONNECTION_TIMED_OUT | Lightsail 방화벽 포트 80/443 미오픈 | Lightsail 콘솔에서 포트 추가 |
| certbot www 인증서 실패 | www DNS 레코드 없음 (NXDOMAIN) | www 빼고 apex만 발급 |
| git push 거절 (fetch first) | 팀원이 동시에 push | git pull 후 재push (2회 발생) |
| file 모드 빈 차트 | data/ gitignore로 서버에 파일 없음 | 데이터 git 포함 + demo 폴백 추가 |
