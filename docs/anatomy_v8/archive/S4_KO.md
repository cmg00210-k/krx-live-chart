> **Stage 색상:** Deep Violet `#2D1B4E`
>
> 본 장은 제3장에서 산출된 이론적 계산 결과가 어떻게 차트 위의 시각 요소로
> 변환되는지를 논증한다. 색상, 형태, 계층, 밀도 제한의 설계 근거는

## 4.1 렌더링 엔진: TradingView Lightweight Charts v5.1.0

CheeseStock은 2,700개 이상의 종목에 대해 최대 13개 그리기 계층을 동시 렌더링한다.
Canvas2D 기반의 TradingView Lightweight Charts(LWC)를 채택한 이유는 다음과 같다.

| 엔진 | 장점 | 단점 | 판정 |
|------|------|------|------|
| SVG | DOM 접근 가능, CSS 스타일링 | O(n) DOM 노드 → 1,000개 이상에서 성능 저하 | 기각 |
| WebGL | GPU 가속, 대량 처리 | 셰이더 파이프라인 복잡, 2D 차트에 과잉 | 기각 |
| Canvas2D | 빠른 래스터화, 간결한 API, DPR 제어 | 히트 테스팅 불가, 수동 텍스트 레이아웃 | **채택** |

LWC의 `ISeriesPrimitive` API를 통해 차트 캔버스 위에 직접 그리기가 가능하며,
이것이 패턴, 신호, 예측 구간의 렌더링 기반이다.

## 4.2 9개 그리기 계층 (PatternRenderer)

PatternRenderer는 고정 순서의 9개 계층 아키텍처를 사용한다. 각 계층은
제3장의 특정 출력 유형을 시각화하며, 후순위 계층이 선순위 위에 그려진다.

| 순서 | 계층명 | 시각 요소 | 제3장 출력 | 색상 |
|------|--------|----------|-----------|------|
| 1 | 글로우(Glows) | 개별 캔들 수직 줄무늬 | 단일 캔들스틱 패턴 | 보라 0.12 |
| 2 | 브래킷(Brackets) | 2-3개 캔들 둥근 사각형 | 이중/삼중 패턴 | 보라 0.12 |
| 3 | 추세영역(TrendAreas) | 그라데이션 다각형 + 피벗 마커 | 삼각형/쐐기형 차트패턴 | 민트 0.12 |
| 4 | 폴리라인(Polylines) | 피벗점 연결선 (W/M/넥라인) | 이중바닥/천정 | 민트 0.65 |
| 5 | 수평선(Hlines) | 지지/저항, 손절/목표 수평선 | S/R 클러스터링, 패턴 목표가 | 은색/오렌지/민트 |
| 6 | 커넥터(Connectors) | H&S 빈 원 + 어깨 연결선 | 머리어깨 피벗점 | 민트 |
| 7 | 라벨(Labels) | 알약형 배지 (패턴명, 한국어) | 모든 감지 패턴 | 흰색/어두운 배경 |
| 8 | 예측구간(ForecastZones) | 목표/손절 그라데이션 + R:R 바 | 패턴 목표가/손절가 | 민트/오렌지 |
| 9 | 연장선(ExtendedLines) | 화면 밖 구조선 연장 | 추세선/넥라인 | 골드 점선 [8,4] |

## 4.3 색채 이론과 설계 근거

### 4.3.1 한국 시장 색상 관례

한국 주식시장은 서양과 **반대** 색상 관례를 따른다:

| 방향 | 한국 (KRX) | 서양 (NYSE) | 근거 |
|------|-----------|------------|------|
| 상승/매수 | **적색** `#E05050` | 녹색 | 동아시아 문화에서 적색은 번영과 길조를 상징 |
| 하락/매도 | **청색** `#5086DC` | 적색 | 청색은 안정과 보수적 태도를 상징 |

이는 미적 선호가 아닌 **문화적 관례**이다. 삼성증권, 미래에셋, NH투자증권,
키움증권 등 모든 한국 트레이딩 플랫폼이 이 관례를 따르며,
CheeseStock도 사용자 기대와 일치시킨다.

### 4.3.2 3열 색상 독립성

CheeseStock의 4열 레이아웃에서 각 열은 독립적인 색상 의미 체계를 갖는다:

| 열 | 영역 | 색상 체계 | 의미 |
|----|------|----------|------|
| B (차트) | 가격 움직임, 지표 | 적색/청색 | 가격 **방향** (상승/하락) |
| C (패턴) | 패턴 주석 | 민트/보라 | 분석 **유형** (차트/캔들) |
| D (재무) | 펀더멘털 지표 | 녹색/청색 | 재무 **품질** (양호/부진) |

**분리 근거:** 패턴은 **방향 중립적**이다. 해머는 지지선에서 출현하든(강세)
저항선에서 출현하든(신뢰도 낮음) 해머이다. 패턴에 방향적 적색/청색을 사용하면
패턴 이론이 뒷받침하지 않는 인지적 편향을 유발한다.

### 4.3.3 패턴 색상 통일

매수와 매도 패턴 모두 **동일한 민트 색상**을 사용한다:

```javascript
PTN_BUY:  'rgba(150,220,200,0.65)',   // 민트
PTN_SELL: 'rgba(150,220,200,0.65)',   // [통일] 매도도 민트
```

**근거:** Bloomberg/TradingView의 전문가 표준을 따른다. 패턴 감지는
**중립적 분석 관찰**이지 방향적 추천이 아니다. 방향 정보는 라벨 텍스트와
위치(가격 위/아래)로 전달한다.

## 4.4 캔버스 위의 타이포그래피

| 폰트 | 용도 | 선정 근거 |
|------|------|----------|
| **Pretendard** 12px 700 | 패턴 라벨, 한국어 텍스트 | 한국어 최적화 가변 폰트, 12px에서도 일관된 자폭 유지 |
| **JetBrains Mono** | 가격 라벨, 종목코드 | 표 형식 숫자(`tnum`)로 소수점 정렬 보장 |

라벨 충돌 회피 알고리즘: 인접 캔들에 복수 패턴이 감지되면 라벨이 겹친다.
정렬 → 겹침 검사 → 수직 오프셋(위→아래 또는 상하 이동) → 최대 3개 제한.

## 4.5 밀도 제어와 인지 부하

### 4.5.1 밀도 제한 상수

| 상수 | 값 | 근거 |
|------|---|------|
| MAX_PATTERNS | 3 | Miller(1956): 작업 기억 7±2 항목. 패턴 3개 × 5 시각요소 = 15개, 이미 인지 한계 |
| MAX_EXTENDED_LINES | 5 | 다수 역사적 패턴의 선 어수선함 방지 |
| MAX_DIAMONDS | 6 | 최근 신호에 집중 |
| MAX_STARS | 2 | 고신뢰 복합 신호 — 설계상 희소 |
| MAX_DIV_LINES | 4 | RSI/MACD 다이버전스선 — 구조적, 비과밀 |
| RECENT_BAR_LIMIT | 50 | 시간적 집중: 최근 ~50봉의 분석만 렌더링 |

### 4.5.2 학술적 근거: 인지 부하 이론

George Miller(1956)의 "마법의 숫자 7, ±2"는 인간의 작업 기억이 동시에
7±2개의 정보 청크를 보유할 수 있음을 확립했다.

차트 맥락에서:
- 패턴 1개 ≈ 5개 시각 요소 (글로우/브래킷 + 라벨 + S/R선 + 예측구간)
- 패턴 3개 × 5 = 15개 시각 요소
- 여기에 캔들스틱, 지표 오버레이, 축 라벨을 더하면 → 인지 용량 포화

**설계 결정:** 모든 감지 패턴 대신 최근/최고 신뢰도 3개만 표시한다.
분석 완전성(제3장이 전체 데이터에 대해 실행)은 보존되며,
시각적 필터링(제4장)이 인간 지각의 한계를 존중한다.

### 4.5.3 시각화 토글과 연산-표시 분리

4개 토글 범주: 캔들 / 차트 / 신호 / 예측

**아키텍처 원칙:** `_filterPatternsForViz()`는 렌더링 시점에 필터링한다.
제3장의 분석은 토글과 무관하게 항상 완전히 실행된다. 따라서:
- 패턴 감지 정확도는 표시 설정에 독립적이다
- 사용자는 재분석 없이 표시를 전환할 수 있다
- 백테스트 결과는 시각화 상태와 무관하게 유효하다

## 4.6 캔버스 DPR(장치 픽셀 비율) 안전성

고해상도 디스플레이(Retina, 4K)에서는 좌표계 스케일링이 필요하다:

```javascript
// 올바른 방법: 스케일링 전 변환 초기화
ctx.setTransform(1, 0, 0, 1, 0, 0);  // 항등행렬로 리셋
ctx.clearRect(0, 0, width, height);
ctx.scale(dpr, dpr);                   // DPR 스케일링 적용
```

리셋 없이 스케일링하면 매 리드로우마다 좌표가 기하급수적으로 증가한다
(2배 → 4배 → 8배...). 이는 미묘한 버그로 나타나며,
보이지 않거나 위치가 어긋나는 그리기로 현현한다.

## 4.7 신호 렌더러 — 이중 패인뷰 아키텍처

| 패인 | zOrder | 시각 요소 | 근거 |
|------|--------|----------|------|
| **배경** | `'bottom'` | 수직 밴드 (골든/데드 크로스 영역) | 맥락 신호가 가격 동작을 가리지 않아야 함 |
| **전경** | `'top'` | 다이아몬드, 별, 다이버전스선, 거래량 라벨 | 고신뢰 신호는 가격 위에 반드시 노출 |

골든/데드 크로스 영역은 다수 봉에 걸쳐 확장되어 전경에 렌더링하면
캔들스틱 패턴을 완전히 가릴 수 있다. 반면 다이아몬드/별 마커는
특정 봉의 점(point) 신호로, 캔들과 공존할 수 있을 만큼 작다.

## 4.8 제3장 → 제4장 매핑표

| 제3장 출력 유형 | 제4장 계층 | 시각적 부호화 | 예시 |
|----------------|-----------|-------------|------|
| 지표값 (MA, BB) | 차트 오버레이 | 색상 선 | MA5 = 적색 선 |
| 캔들 패턴 | 계층 1-2 + 계층 7 | 보라 하이라이트 + 배지 | "해머" 배지 |

| S/R 수준 | 계층 5 | 은색 수평선 + 가격 라벨 | 50,000원 지지선 |
| 신호 | 신호 렌더러 전경 | 다이아몬드(중) 또는 별(강) | 골든크로스 다이아몬드 |
| 예측 구간 | 계층 8 | 민트/오렌지 그라데이션 | 목표/손절 투영 |

| 백테스트 결과 | 패턴 패널 카드 (C열) | 승률 %, 평균 수익률 % | "승률 62%" 텍스트 |
