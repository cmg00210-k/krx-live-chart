# 01. 학술 문서(core_data/) <-> 패턴 엔진(patterns.js) 연계 매핑

> 각 core_data 문서가 어떤 패턴/지표의 이론적 근거를 제공하는지,
> 그리고 현재 구현(js/)과의 갭을 체계적으로 분석한다.

## 2. 역방향 매핑: 구현 -> 학술 근거

### 구현되었으나 학술 근거가 약한 항목

| 구현 항목 | 코드 위치 | 문제 |
|----------|----------|------|
| `_quality()` 가중치 | L73-76 | w1~w5 = (0.25, 0.25, 0.20, 0.15, 0.15) -- 이론적 근거 미명시. 07 S3.6에서 백테스팅 기반 캘리브레이션 권장하나 미적용 |
| `_target()` 배수 1.5x | L87-94 | 패턴 높이 * 1.5 -- Bulkowski 측정 이동(Measured Move) 기준과 다름. 표준은 1.0x |
| `_stopLoss()` ATR 2배 | L79-84 | 2x ATR 기본값 -- 07에서는 패턴별 차등 권장 |
| `_detectTrend()` 임계값 0.3 | L52 | norm > 0.3 → 상승 판정 -- 근거 미명시 |
| 도지 body < range*0.05 | L388 | Nison 기준은 0.05~0.10 범위 -- 현재 0.05로 엄격 |

### 구현되었으나 학술 문서에 기술되지 않은 항목
