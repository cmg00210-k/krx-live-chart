### 2.2 구현된 기능 목록

**동작하는 기능:**
- 캔들스틱/라인/바 차트 (TradingView Lightweight Charts v4.2.3)
- 27종 캔들/차트 패턴 인식 (ATR 정규화, 품질 점수 0-100)
- 28종 지표 시그널 (MA/MACD/RSI/BB/Volume) + 복합 시그널 6종
- 기술적 지표 8종 (MA, EMA, BB, RSI, MACD, Ichimoku, Kalman, 거래량)
- 패턴/시그널 Canvas2D 시각화 (ISeriesPrimitive)
- 패턴 호버 툴팁 (크로스헤어 연동)
- 드래그 기반 패턴 실시간 분석
- 패턴 요약 바 + 상세 팝업 (차트 하단)
- 패턴 과거 수익률 바 (5일 후 수익률, 승률)
- 시그널 카테고리 필터 (6카테고리 체크박스)
- 종목 검색 (이름/코드, 2,700+ 종목)
- 접이식 사이드바 (시가총액순, 55개 DEFAULT_STOCKS)
- 우측 재무지표 패널 (DART + 하드코딩 폴백)
- 영업이익률 스파크라인
- Web Worker 비동기 분석 (메인 스레드 비차단)
- 실시간 WebSocket 연동 (Kiwoom OCX 서버)
- OHLC 크로스헤어 정보 바
- 가격 변화 Flash 애니메이션
- 로딩 Shimmer 피드백
- 반응형 레이아웃 (768px/1024px/1200px)

**아직 동작하지 않는 기능 (서버 의존):**
- Kiwoom OCX 서버가 실행되지 않으면 WS 모드 작동 안 함 → DEMO 모드로 정적 차트 표시
- 분봉 데이터: file 모드에서 데모 폴백 (WS 모드에서만 실제 분봉)
- KOSPI/KOSDAQ 실시간 지수: WS 서버의 `market_index` 메시지 필요
- 실시간 체결 틱 (SetRealReg): 서버 측 구현 필요

### 2.3 Kiwoom 서버 연결 상태

현재 `KRX_API_CONFIG.mode = 'ws'` (기본값)로 설정.
서버가 없으면:
1. WebSocket `ws://localhost:8765` 연결 시도
2. 최대 10회 재연결 (지수 백오프, 최대 30초)
3. 연결 실패 시 `OFFLINE` 상태 표시 (데모 폴백 없음)
4. `data/` 폴더에 JSON 파일이 있으면 file 모드로 전환 가능

### 데이터 흐름 (Data Flow)

```
1. 종목 선택 → selectStock(code)
2. dataService.getCandles(stock, timeframe)
   ├─ ws 모드: 빈 배열 반환 → realtimeProvider에서 서버 캔들 수신
   ├─ file 모드: data/{market}/{code}.json 로드
   └─ demo 모드: 시드 기반 시뮬레이션
3. candles[] 수신 → updateChartFull()
   ├─ Worker 분석 요청 (비동기)
   │   ├─ patternEngine.analyze(candles) → patterns[]
   │   └─ signalEngine.analyze(candles, patterns) → signals[], stats
   ├─ chartManager.updateMain(candles, chartType, indicators, patterns)
   │   ├─ 지표 시리즈 업데이트 (MA/EMA/BB/Ichimoku/Kalman)
   │   ├─ patternRenderer.render() → Canvas2D 패턴 도형
   │   └─ signalRenderer.render() → Canvas2D 시그널 마커
   ├─ chartManager.updateRSI(candles) (RSI 활성 시)
   └─ chartManager.updateMACD(candles) (MACD 활성 시)
4. 드래그 → onVisibleRangeChange → 보이는 구간만 재분석 (메인 스레드)
5. 호버 → subscribeCrosshairMove → 패턴/시그널 툴팁
```

### 5.2 기술 스택 로드맵

| 레이어 | 현재 | 목표 |
|--------|------|------|
| 프론트엔드 | index.html + JS (빌더 없음) | Vite + TypeScript (선택적) |
| 차트 | TradingView LWC v4.2.3 | 동일 (안정적) |
| 서버 | Python 3.9-32bit + PyQt5 (Kiwoom) | FastAPI + asyncio |
| 데이터 소스 | Kiwoom OCX (개발) | Koscom API (운영) |
| ML | XGBoost (heartstock 프로젝트) | LightGBM/Catboost + SHAP 설명 |
| 데이터베이스 | JSON 파일 | PostgreSQL + TimescaleDB |
| 배포 | 로컬 | Docker + nginx + SSL |

### 5.3 Kiwoom → Koscom 전환 로드맵

1. **현재**: Kiwoom OCX → Python WebSocket 서버 → 브라우저
   - 장점: 무료, 32-bit 제약 가능
   - 단점: Windows 전용, OCX 의존, 개인 투자자 한정

2. **전환 계획**: DataProvider 추상화 계층 (이미 설계됨)
   - `KiwoomProvider`: 현재 구현 (kiwoom_project)
   - `KoscomProvider`: Koscom 오픈 플랫폼 API
   - `PykrxProvider`: 장외 시간 폴백

3. **Koscom 스타트업 프로그램**:
   - 설립 7년 미만 기업: 3년간 API 무료 사용
   - 실시간 시세 + 호가 + 체결 데이터 포함
   - REST/WebSocket 표준 API (OCX 불필요)

### 5.4 서비스 비전

**"기술적 분석 + AI 패턴 + 리서치 보고서 플랫폼"**

1. **기술적 분석 차트**: 현재 구현의 웹 버전 (누구나 접근 가능)
2. **AI 패턴 예측**: heartstock 프로젝트의 ML 파이프라인 통합
3. **자동 리서치 보고서**: 종목 스캔 → 패턴 감지 → 수익률 통계 → PDF 리포트
4. **실시간 알림**: 특정 패턴/시그널 발생 시 푸시 알림
5. **포트폴리오 분석**: 보유 종목 기반 리스크 분석, 상관관계 히트맵
